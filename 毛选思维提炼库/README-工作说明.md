## 七、注意事项

- **语言风格**：极其接地气、商业化、犀利，禁止政治课本术语
- **核心原则**：忽略历史细节（战役、人名、年代），只提炼底层思维
- **模型命名**：用带引号的中文短语，如「利益分层定位法」
- **批量处理**：每轮对话建议处理 3-5 篇（双线产出每篇约 2000-3000 token，含原文读取），处理完后主动停止并报告进度。**宁可多开一轮对话，也不要硬撑到上下文爆掉——爆掉意味着最后一篇的产出可能丢失**
- **断点保护**：每完成一篇，立即写入文件。不要攒着批量写入，防止上下文溢出导致全军覆没
- **进度同步**：每轮结束时，更新本文件 frontmatter 的 `completed` 和 `next` 字段，确保下次对话能精准续接
- **文件命名**：产出文件名与原文文件名完全一致
- **双线产出**：每篇必须同时完成产线一（报告）和产线二（skill XML），缺一不可
- **象限判定**：每篇必须明确判定 A/B/C/D 象限，写入产线二开头
- **自动更新工作说明**：每轮结束时，必须同步更新本文件三处——① frontmatter 的 `completed`/`next`/`updated` 字段 ② 第六节进度追踪表 ③ 第八节快速恢复指令中的编号。确保用户在新对话中只需发送一条指令即可无缝续接
