# maoxuan-mindset

> 将《毛泽东选集》全部 **229 篇原文**，逐篇进行「跨领域思维迁移」式提炼，产出可复用的战略思维模型，并封装为 AI Agent 可调用的决策引擎。

<p align="center">
  <strong>229 篇原文</strong> · <strong>229 份思维报告</strong> · <strong>229 条决策算法</strong> · <strong>4 大象限</strong>
</p>

---

## 这是什么

本仓库包含三个核心模块：

| 模块 | 说明 |
|------|------|
| 📖 **毛选原文** | 全部 229 篇原文，Markdown 格式，来自 [weiyinfu/MaoZeDongAnthology](https://github.com/weiyinfu/MaoZeDongAnthology) |
| 🧠 **思维提炼库** | 每篇原文对应一份「五模块」思维提炼报告，提炼底层思维模型并映射到现代商业/职场/个人发展场景 |
| ⚡ **Skill 决策引擎** | 将 229 篇精华封装为「诊断漏斗 + 四象限算法库」，可被 AI Agent 直接调用，输出可执行的诊断书 |

**一句话概括**：不谈主义、不讲历史细节，只提炼「底层思维模型」——毛选里那些真正能帮你破局的东西。

---

## 项目亮点

- **完整覆盖**：毛选 229 篇原文，一篇不少
- **双线产出**：每篇同时产出人类可读的思维报告 + AI 可执行的算法条目
- **模型命名**：所有思维模型都有精炼中文名称，如「利益分层定位法」「窗口期压倒性收割模型」「使命锚定术」
- **即插即用**：Skill 决策引擎可直接集成到 Claude / GPT / 任意 AI Agent 中使用
- **实战导向**：每篇包含现代场景映射 + 行动指导 SOP，不是读完后「觉得有道理但不知道怎么用」

## 每篇报告长什么样

每篇思维提炼报告都包含 **五个标准模块**：

### 模块一：核心算法一句话总结

用一句最具震撼力的现代大白话总结全文精髓。

> *例：「做事之前先搞清楚：谁是你的基本盘，谁是你的敌对阵营，谁是可以被争取的中间票——搞错这一点，所有努力都是送人头。」*
> —— 《中国社会各阶级的分析》

### 模块二：历史困境与破局逻辑（20% 篇幅）

当时的绝境是什么？破局的核心操作是什么？忽略历史细节，只看底层逻辑。

### 模块三：顶层思维模型提取（40% 篇幅，核心）

提炼 2-3 个通用思维模型，并命名。精选模型示例：

| 模型名称 | 来源篇目 | 一句话 |
|---------|---------|--------|
| 「利益分层定位法」 | 中国社会各阶级的分析 | 按利益受损程度切分市场，锁定天然盟友 |
| 「窗口期压倒性收割模型」 | 对日寇的最后一战 | 趋势确认后，在窗口期内集中一切资源压倒性收割 |
| 「使命锚定术」 | 为人民服务 | 用一个超越个体的使命锚定组织，防止内耗和方向漂移 |
| 「三线同时崩盘定律」 | 蒋介石政府已处在全民的包围中 | 当政治、经济、军事三线同时崩溃，崩盘不可逆 |
| 「最坏假设部署法」 | 在中国共产党全国代表会议上的讲话 | 永远按最坏情况做预案，确保任何结果都能接住 |
| 「卸包袱-开引擎」双动模型 | 学习和时局 | 先卸掉历史包袱（骄傲/自卑），再打开引擎（主动学习） |
| 「纪律即品牌」 | 三大纪律八项注意 | 纪律不是约束，是组织对外的品牌承诺 |
| 「有限难题-无限坚持定理」 | 愚公移山 | 有限难度的难题 × 无限时间的坚持 = 必胜 |

### 模块四：现代场景映射

将模型映射到现代商业/大厂内卷/个人副业场景，各举 2 个具体案例。

### 模块五：行动指导 SOP

实战检查清单：3-5 个立刻可以执行的反思问题或具体动作。

## 四象限模型

所有 229 条决策算法按「战略维度」分为四个象限：

```
                    ┌─────────────────────────────────────────────────┐
                    │                                                 │
     认知层         │   A 象限：认知与方向（52 条）                      │
     「看不清」      │   主要矛盾 · 调研方法 · 趋势判断 · 方向选择        │
                    │                                                 │
                    ├─────────────────────────────────────────────────┤
                    │                                                 │
     关系层         │   B 象限：利益与阵营（69 条）                      │
     「搞不定人」    │   敌友识别 · 统一战线 · 谈判博弈 · 利益分配        │
                    │                                                 │
                    ├─────────────────────────────────────────────────┤
                    │                                                 │
     组织层         │   C 象限：组织与管理（63 条）                      │
     「团队不行」    │   领导力 · 思想统一 · 动员 · 反内耗 · 纪律        │
                    │                                                 │
                    ├─────────────────────────────────────────────────┤
                    │                                                 │
     执行层         │   D 象限：战术与执行（45 条）                      │
     「知道但做不到」│   集中优势 · 以弱对强 · 持久战 · 根据地 · 实操    │
                    │                                                 │
                    └─────────────────────────────────────────────────┘
```

**象限判断速查：**

| 你的困境 | 归属象限 | 典型关键词 |
|---------|---------|-----------|
| 看不清方向、找不到路、不知从哪下手 | A 认知与方向 | 趋势、判断、调研、矛盾 |
| 搞不清敌友、合作关系混乱 | B 利益与阵营 | 敌人、盟友、利益、谈判 |
| 团队内耗、执行力差、思想不统一 | C 组织与管理 | 团队、领导力、动员、纪律 |
| 知道该干但不知道怎么干、以弱对强 | D 战术与执行 | 打法、资源少、竞争、持久战 |

## Skill 决策引擎

### 工作原理

决策引擎采用「诊断漏斗」四步流程：

```
用户描述困境
    │
    ▼
Step 1：象限分类 ──→ 判断属于 A / B / C / D 哪个象限
    │
    ▼
Step 2：加载算法库 ──→ 读取对应象限的算法参考文件
    │
    ▼
Step 3：条件匹配 ──→ 在 233 条条件索引中匹配困境类型，找到对应算法
    │
    ▼
Step 4：输出诊断书 ──→ 输出战略意图 + 动作步骤 + 核心模型 + 行动建议
```

### 文件结构

```
skill/
├── skill.md                        ← 入口文件（~40KB）
│                                      包含：诊断漏斗 + 4 张条件索引表
│
├── quadrant-A-认知与方向.md          ← A 象限算法库（~57KB，52 条算法）
├── quadrant-B-利益与阵营.md          ← B 象限算法库（~75KB，69 条算法）
├── quadrant-C-组织与管理.md          ← C 象限算法库（~71KB，63 条算法）
└── quadrant-D-战术与执行.md          ← D 象限算法库（~48KB，45 条算法）
```

每条算法的结构：

```markdown
## ALGO_XXX | 核心思维名称

**战略意图：** 一句话描述

**动作步骤：**
1. [具体动作 1]
2. [具体动作 2]
3. [具体动作 3]

**核心模型：** 模型一、模型二、模型三

**来源：** [[毛选思维提炼库/XXX-文章名]]
```

### 如何使用

#### 方式一：直接阅读

1. 打开 `毛选思维提炼库/skill/skill.md`，浏览四张条件索引表
2. 找到与你困境匹配的关键词，记下对应的算法编号（如 `ALGO_017`）
3. 打开对应象限文件，搜索算法编号，查看完整的动作步骤

#### 方式二：AI Agent 集成

将 `skill.md` 作为 System Prompt 加载给 AI Agent，即可实现：
- 用户描述困境 → Agent 自动判断象限 → 匹配算法 → 输出诊断书

适配 Claude / GPT / 任意支持长上下文的 AI 工具。核心流程只需 3 步：

1. 将 `skill/skill.md` 加载为系统提示词
2. 用户输入困境描述
3. Agent 按诊断漏斗执行，按需加载对应象限文件
